Source Code

These source-code files were transcribed from scans made from Don Eyles's personal copy of Luminary 069. They were scanned at archive.org's Boston facility, and the scanning was sponsored by Onno Hommes. The code was transcribed from these scans by a team of volunteers who are referenced in the program comments. Comments from the original source code are prefixed with a single '#' symbol, whereas comments added later are prefixed by "##" or "###". In some cases, where similar code blocks exist in previously-transcribed AGC programs (primarily Luminary 99, from Apollo 11) those code blocks were used as a starting point and then corrected to agree with the Luminary 69 scans. The full scans are available at the Virtual AGC project's collection at archive.org, while more-convenient reduced-size (but reduced-quality) images are available at the main Virtual AGC website. Report any errors noted by creating an issue report at the Virtual AGC project's GitHub repository. Notations on the program listing read, in part:

	GAP:  ASSEMBLE REVISION 069 OF AGC PROGRAM LUMINARY BY NASA 2021112-011
	19:02 NOV. 25,1968
Note that the date is the date of the printout, not the date of the program revision.

004674,000034:                                                                                                  #  CONVENTIONS AND NOTATIONS UTILIZED FOR ERASABLE ASSIGNMENTS.
004675,000035: 
004676,000036:                                                                                                  #           EQUALS IS USED IN TWO WAYS.  IT IS OFTEN USED TO CHAIN A GROUP
004677,000037:                                                                                                  #                  OF ASSIGNMENTS SO THAT THE GROUP MAY BE MOVED WITH THE
004678,000038:                                                                                                  #                  CHANGING OF ONLY ONE CARD.  EXAMPLE.
004679,000039: 
004680,000040:                                                                                                  #                                          X        EQUALS  START
004681,000041:                                                                                                  #                                          Y        EQUALS  X       +SIZE.X
004682,000042:                                                                                                  #                                          Z        EQUALS  Y       +SIZE.Y
004683,000043: 
004684,000044:                                                                                                  #                  (X, Y, AND Z ARE CONSECUTIVE AND BEGIN AT START.       )
004685,000045:                                                                                                  #                  (SIZE.X AND SIZE.Y ARE THE RESPECTIVE SIZES OF X AND Y,
004686,000046:                                                                                                  #                   USUALLY NUMERIC,  IE. 1, 2, 6, 18D ETC.                )
004687,000047: 
004688,000048:                                                                                                  #           EQUALS OFTEN IMPLIES THE SHARING OF REGISTERS (DIFFERENT NAMES
004689,000049:                                                                                                  #                  AND DIFFERENT DATA).  EXAMPLE.
004690,000050: 
004691,000051:                                                                                                  #                                          X        EQUALS Y
004692,000052: 
004693,000053:                                                                                                  #           = MEANS THAT MULTIPLE NAMES HAVE BEEN GIVEN TO THE SAME DATA.
004694,000054:                                                                                                  #                  (THIS IS LOGICAL EQUIVALENCE, NOT SHARING)  EXAMPLE.
004695,000055: 
004696,000056:                                                                                                  #                                          X        =      Y
004697,000057: 
004698,000058:                                                                                                  #           THE SIZE AND UTILIZATION OF AN ERASABLE ARE OFTEN INCLUDED IN
004699,000059:                                                                                                  #           THE COMMENTS IN THE FOLLOWING FORM.  M(SIZE)N.
004700,000060: 
004701,000061:                                                                                                  #                  M REFERS TO THE MOBILITY OF THE ASSIGNMENT.
004702,000062:                                                                                                  #                       B   MEANS THAT THE SYMBOL IS REFERENCED BY BASIC
004703,000063:                                                                                                  #                           INSTRUCTIONS AND THUS IS E-BANK SENSITIVE.
004704,000064:                                                                                                  #                       I   MEANS THAT THE SYMBOL IS REFERENCED ONLY BY
004705,000065:                                                                                                  #                           INTERPRETIVE INSTRUCTIONS, AND IS THUS E-BANK
004706,000066:                                                                                                  #                           INSENSITIVE AND MAY APPEAR IN ANY E-BANK.
004707,000067: 
004708,000068:                                                                                                  #                  SIZE IS THE NUMBER OF REGISTERS INCLUDED BY THE SYMBOL.
004709,000069: 
004710,000070:                                                                                                  #                  N INDICATES THE NATURE OR PERMANENCE OF THE CONTENTS.
004711,000071:                                                                                                  #                       PL  MEANS THAT THE CONTENTS ARE PAD LOADED.
004712,000072:                                                                                                  #                       DSP MEANS THAT THE REGISTER IS USED FOR A DISPLAY.
004713,000073:                                                                                                  #                       PRM MEANS THAT THE REGISTER IS PERMANENT, IE. IT
004714,000074:                                                                                                  #                           IS USED DURING THE ENTIRE MISSION FOR ONE
004715,000075:                                                                                                  #                           PURPOSE AND CANNOT BE SHARED.
004716,000076:                                                                                                  #                       TMP MEANS THAT THE REGISTER IS USED TEMPORARILY OR
004717,000077:                                                                                                  #                           IS A SCRATCH REGISTER FOR THE ROUTINE TO WHICH
004718,000078:                                                                                                  #                           IT IS ASSIGNED.  THAT IS, IT NEED NOT BE SET
004719,000079:                                                                                                  #                           PRIOR TO INVOCATION OF THE ROUTINE NOR DOES IT
004720,000080:                                                                                                  #                           CONTAIN USEFUL OUTPUT TO ANOTHER ROUTINE.  THUS
